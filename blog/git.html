Getting Started

Git, Github & The Command Line

Feb 15, 2016

So, you’re a fresh-faced, soon-to-be developer and you’re ready to start banging out your first lines of code. Maybe you want to rip out a “Hello, World” or a fizzbuzz to get started? Not so fast! The most important thing you can do when you’re first starting out as a code monkey is to get comfortable with your development environment. Until this point, you’ve probably been using a GUI (Graphical User Interface) to get things done on a computer. Mostly you’ve been clicking, dragging and scrolling to use your computer’s services, but there is another way. A better way. A faster way: The command line.

Your computer’s terminal is the same thing as your regular computer, it gives you access to all of your computer’s same services but instead of having to move your mouse around and navigate through windows to get this done, you use short, concise text commands. In many cases, there are operations that would be quite time consuming if you tried to do them through a GUI, but only take a few lines of Bash. What’s Bash, you ask? Bash is the language that your shell speaks. What is your shell, you ask? Technicalities aside, your shell is your terminal and also is your command line. They are all the same thing. Now open it up and let’s get started! On a mac, open the program called Terminal. On Windows or linux or anything else you well, you google how to use your terminal, because I don’t use those machines and I’m too lazy to figure out how best to access your shell. Plus, you should at Googling things anyway. Google holds all the answers.

Awesome! Now you’re in your terminal and you can start talking to your computer. It should look like you’re using a computer from the early 90s.

  • Type pwd. This stands for “print working directory” and it will give you a filepath to the directory you’re in. It’s probably something like Users/yourusername. Cool, that’s where you are. Quick aside, a directory is the same as a folder.
  • Type ls. This lists all of the folders or files that are in your working directory. You probably have a folder called Desktop, Application, Downloads.
  • OK, I want to start working on a new folder in my Desktop.
  • Type cd Desktop. This stands for Change Directory and the name that comes after it is the directory you want to navigate to. Using cd this way, you can’t jump around from any folder to any other folder. You can only move down the folder tree to one of the folder’s children.
  • Type cd .. , this will navigate you up the folder tree one folder. Now type in cd Des but before you finish typing desktop, hit the tab key. DId you see it finish the word for you? This is called tab completion and it’s handy because it’s fast and also shows you which folders you can navigate to.
  • Now that you’re in your Desktop folder we want to start a new project. We’ll call it, my_first_project.
  • mkdir my_first_project. mkdir stands for Make Directory and then you tell it what you want that directory to be called.
  • cd my_first_project, to move into that folder.
  • touch hello_world.html. Touch creates an empty file.
  • You now have all of the basics you need for moving around your computer in terminal and creating new folders and files!

You might be thinking, this isn’t faster, this is a lot lot slower! Give it some time. You’ve probably been using a GUI for many years and the terminal is probably very new to you. Once you get over the learning curve, the terminal will speed up your development time immensely.

Now that you’ve set up your first working folder, it’s time to start using Git. Git is a version control system that saves a version of your working directory (also called repository in git world, I know so many names for the same thing) at a certain point in time. You save these versions by making a commit. A commit is a save point. It is a snapshot of the codebase at a specific point in time. Once saved, git offers features to: rollback to that point in time, merge or rebase two saved versions together, navigate through different saved states or sync local changes with a remote repository to name just a few. It’s a good habit to make commits often, and an even better habit to make a commit after every time you create a piece of working code, committing broken code is not ideal. When programming, it’s easy to really mess things up. Git offers you the ability to go back to a point in time when your code was working and start fresh from there.

  • You may already have git installed. To check, run git --version in your terminal. If you see a git version, great! If not, Google how to install git on your machine and install it.
  • Go back to terminal. If you aren’t already in my_first_project, use the commands you learned to navigate inside of it. If you ls, you should see hello_world.html and that’s it.
  • git init. This starts up git on your repository. This installs a hidden folder called .git in my_first_project that will do all of the magic for you.
  • git status. This will tell you the status of your repository. RIght now you should see that there are untracked changes to hello_world.html
  • git add -A. This will add hello_world.html to your staging area, this is saying that you are ready to commit.
  • git commit -m “first commit for my first project”. This creates a save point. You have to write a message with your save point, that’s what the -m flag does. These messages are short concise statements that tell you what is happening in the commit.

Great! You’ve saved your code! But what happens if your computer accidentally explodes? Github to the rescue! Github is your remote repository, remote meaning on someone else’s computer, not yours. It allows you to save your work in the cloud and collaborate with other people who are also doing the same thing.

  • First set up a github account if you don’t have one already.
  • Now, select the plus symbol on the top righthand side of the screen and select “New repository.”
  • Now follow the steps and create a repository called my_first_project.
  • Once that’s created, grab the url next to the http drop starting with http://github.com/...
  • Go back to terminal and in your working directory type git remote add origin [the url you just grabbed]. It may ask you to provide your login credentials.
  • Now your local repo is connected to your remote repo.
  • Now you want to save the changes up to the cloud for safe keeping and sharing.
  • git push origin master. This will push the last commit up to the cloud.

And there you have all of your working code stored in the cloud! This is a simplistic example but there are lots more things you can do with git and Github. People who work on teams use git clone to get a repository, git pull to update their local repo, pull requests are also used for submitting code for review so feature branches can be merged with the master branch. Git allows you to fork code merge or rebase depending on your team’s development style. If you’re curious about any of this, Google “best git tutorial” or “best github tutorial” and go from their.

With this knowledge, you are now ready to start coding! Your code will be safe and your efficiency will be high with knowledge of the command line, git and github.
